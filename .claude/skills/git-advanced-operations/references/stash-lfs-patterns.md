# Stash and LFS Patterns

## Stash Basics

A stash saves your working directory and index state as temporary commits
on a special ref stack (`refs/stash`).

```bash
# Stash all tracked modifications and staged changes
git stash

# Always use descriptive messages
git stash push -m "WIP: auth middleware refactor"

# Stash only staged changes (keep unstaged work)
git stash push --staged -m "staged: API response format"

# Stash including untracked files
git stash push --include-untracked -m "with new migration files"

# Stash everything including ignored files
git stash push --all -m "full workspace snapshot"
```

## Partial Stash

```bash
# Stash only specific paths
git stash push -m "auth changes only" -- src/auth.py src/middleware.py

# Patch mode: choose hunks interactively
git stash push --patch -m "selected changes"
# y = stash this hunk, n = skip, s = split, e = edit, q = quit
```

## Listing and Inspecting Stashes

```bash
# List all stashes
git stash list

# Show stash diff summary
git stash show stash@{0}

# Show full diff (patch)
git stash show -p stash@{0}

# Show only file names
git stash show --name-only stash@{0}

# Show untracked files in a stash
git show stash@{0}^3  # Third parent holds untracked files
```

## Applying and Popping Stashes

```bash
# Pop: apply and remove from stack (most common)
git stash pop

# Pop a specific stash
git stash pop stash@{2}

# Apply: apply but keep on the stack (useful for applying to multiple branches)
git stash apply stash@{0}

# Apply and restore the index (staging area) state
git stash apply --index stash@{0}

# If pop/apply has conflicts:
# 1. Resolve the conflicts
# 2. The stash is NOT dropped on conflict (even with pop)
# 3. Drop manually after resolving:
git stash drop stash@{0}
```

## Creating a Branch from a Stash

```bash
# Create a new branch and apply the stash to it
git stash branch feature/from-stash stash@{0}

# This is equivalent to:
# git checkout -b feature/from-stash <commit-where-stash-was-created>
# git stash pop stash@{0}

# Useful when:
# - Stash conflicts with current branch changes
# - You realize stashed work deserves its own branch
# - You stashed on the wrong branch
```

## Dropping and Clearing Stashes

```bash
# Drop a specific stash
git stash drop stash@{1}

# Clear ALL stashes (cannot be undone easily)
git stash clear

# Before clearing, list what you have:
git stash list
```

## Recovering Dropped Stashes

```bash
# Stashes are commits. Dropped stashes become dangling commits.
git fsck --no-reflogs | grep "dangling commit"

# Inspect each to find your stash, then apply it
git show <hash>
git stash apply <hash>
```

## Stash Workflow Patterns

```bash
# Quick context switch
git stash push -m "WIP: feature-x halfway done"
git checkout hotfix-branch
# ... fix, commit, push ...
git checkout feature-x
git stash pop

# Testing a clean state
git stash push --include-untracked -m "testing clean build"
make test
git stash pop

# Moving changes to a different branch
git stash push -m "work meant for feature branch"
git checkout -b correct-feature-branch
git stash pop
```

## Git LFS (Large File Storage)

Git LFS replaces large files in your repository with tiny pointer files,
storing the actual content on a separate LFS server.

## LFS Setup

```bash
# Install Git LFS (macOS: brew install git-lfs, Ubuntu: sudo apt install git-lfs)

# Initialize LFS for your user (one-time)
git lfs install
```

## Tracking Files with LFS

```bash
# Track by file extension
git lfs track "*.psd"
git lfs track "*.zip"
git lfs track "*.bin"

# Track by directory
git lfs track "datasets/**"
git lfs track "models/*.pkl"

# Track specific files
git lfs track "data/large-dataset.parquet"

# IMPORTANT: commit .gitattributes after tracking
git add .gitattributes
git commit -m "chore: configure LFS tracking patterns"

# Now add and commit large files normally
git add data/large-dataset.parquet
git commit -m "feat: add training dataset"
```

## .gitattributes File

LFS tracking rules are stored in `.gitattributes`:

```gitconfig
# .gitattributes (auto-generated by git lfs track)
*.psd filter=lfs diff=lfs merge=lfs -text
*.zip filter=lfs diff=lfs merge=lfs -text
*.bin filter=lfs diff=lfs merge=lfs -text
datasets/** filter=lfs diff=lfs merge=lfs -text
```

## LFS Status and Information

```bash
# See which tracked files have pending changes
git lfs status

# List all LFS files in the repo
git lfs ls-files

# List LFS files with sizes
git lfs ls-files --size

# Show LFS environment and config
git lfs env

# Verify LFS objects
git lfs fsck
```

## Migrating Existing Files to LFS

```bash
# Migrate files already in the repo history to LFS
# WARNING: This rewrites history!

# Migrate a file type across all branches
git lfs migrate import --include="*.bin" --everything

# Migrate specific paths
git lfs migrate import --include="data/" --everything

# Migrate only recent history (less disruptive)
git lfs migrate import --include="*.psd" --above=1mb

# Preview what would be migrated (dry run)
git lfs migrate info --include="*.bin" --everything

# After migration, force push (coordinate with team!)
git push --force-with-lease --all
git push --force-with-lease --tags
```

## Fetching and Pulling LFS Files

```bash
# Fetch LFS objects for current branch
git lfs fetch

# Pull: fetch + checkout (download and replace pointers)
git lfs pull

# Clone WITHOUT LFS files (faster CI)
GIT_LFS_SKIP_SMUDGE=1 git clone https://github.com/org/repo.git
# Then selectively fetch what you need
git lfs pull --include="models/*.pkl"
```

## LFS Maintenance

```bash
# Prune old LFS objects not referenced by recent commits
git lfs prune --dry-run
git lfs prune

# Verify LFS object integrity
git lfs fsck
```

## LFS Locking (Binary File Conflicts)

```bash
# Enable locking for unmergeable file types
git lfs track --lockable "*.psd"

# Lock before editing, unlock after push
git lfs lock assets/design.psd
git lfs unlock assets/design.psd

# See current locks
git lfs locks
```

## LFS Troubleshooting

```bash
# File shows as pointer text instead of content
git lfs checkout path/to/file.bin

# LFS objects missing after clone
git lfs fetch && git lfs checkout

# Reset LFS cache if corrupted
rm -rf .git/lfs/tmp && git lfs fetch
```
